this file describes the APM manifest.json schema

manifest.json has the following format
{
    structure<UIGroup[]>: defines the UI structure
    commands<Record<string,Command>>: defines the commands available to the structure
    payloads<Record<string,CommandPayload>>: defines the command payloads available to the structure
    codefile<string?>: if present, defines a file that PIXL code may be referenced from
}

UIGroup:
{
    zone<string>: the UI zone this group resides in
    name<string>: the name of this group
    components<UIComponent[]>: a list of components that make up this group
}
UIComponent:
{
    type<string>: what UI element this component is
    attrs<Record<string,string>>: attributes to be passed to the frontend UI component during creation (eg. the text of a button)
    id<string?>: optional id for reference by other UI components, must match /[a-zA-Z0-9_-]+/
    routing_key<string|"!"|("."+string)|null>: see Routing Keys
    command_id<string>: the command that is executed, must be a key of the "commands" field of the ancestor Manifest
    payload<Record<string,PayloadCondition>>: the payload that is used, see Payloads
}

PayloadCondition:
{
    priority<non-negative int>: the priority of this condition
    condition<string>: the condition, see Conditions
}

CommandPayload:
Record<string,Record<string,(
    {raw<any>: raw value}|
    {dyn<string>: value derived from a field, see Payloads}
)>>

Command:
{
    func<string>: the actual function that handles this command
    bind<RESERVED>: reserved for future use in binding some arguments
}


Routing Keys:
routing keys are provided to command handlers as the first argument, allowing one handler to handle many related requests (such as discarding changes from different dialogs)
the "routing_key" field of the UIComponent allows for three forms:
"!", null, or omitted - no "routing_key", this component is not interactive
"."+string - auto-generates the rest of the routing key, produces a key of the form "{module}.{zone}.{string}"
string - manual specification of the full routing key

Conditions:
a condition is a PIXL expression (see "docs/pixl.txt")
the resolved value is implicitly coerced to a boolean value using PIXL boolean coercion semantics

Payloads:
payloads are defined in the Manifest.payloads record, a payload field with the "raw" field will be the hardcoded value of the field, a field with the "dyn" field will be derived from another ui component's state

the payload to be used with a command is determined from the payload conditions, it is an error if no condition succeeds
payload conditions are evaluated in order from highest to lowest priority, the first condition that passes will be used

the keys of the UIComponent.payload record must be keys of the Manifest.payloads record, when a condition passes the payload used is the one with the same key

dynamic payload fields are of the form "{component_id}.{field_name}"
the reserved component id of "@self" refers to the component executing the command
